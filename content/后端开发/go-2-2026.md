## Go 2.0：十年来最重要的版本

2026年，Go 2.0正式发布，这是自Go 1.0（2012年）以来最重要的版本更新。Go 2.0保持了Go语言"简单、高效、可靠"的核心哲学，同时在开发者长期呼吁的关键领域进行了重大改进。

根据Go Developer Survey 2026，Go仍然是全球使用率第三高的后端编程语言（仅次于JavaScript/TypeScript和Python），全球有超过360万Go开发者。Go在云原生基础设施、微服务和CLI工具领域占据主导地位。

## 迭代器（Iterators）：Go 2.0的标志性特性

迭代器是Go 2.0最重要的新特性，它填补了Go语言长期以来的一个空白——对自定义集合类型进行优雅的遍历。

### 基本语法

```go
// 定义一个迭代器
func Backward[E any](s []E) func(yield func(int, E) bool) {
    return func(yield func(int, E) bool) {
        for i := len(s) - 1; i >= 0; i-- {
            if !yield(i, s[i]) {
                return
            }
        }
    }
}

// 使用 range 遍历迭代器
for i, v := range Backward([]string{"a", "b", "c"}) {
    fmt.Println(i, v)
}
```

### 标准库中的迭代器

Go 2.0的标准库全面支持迭代器：

- `slices.All`、`slices.Backward`、`slices.Collect`
- `maps.All`、`maps.Keys`、`maps.Values`
- `strings.Lines`、`strings.SplitSeq`
- `bufio.Scanner` 实现了迭代器接口

### 实际影响

迭代器给Go代码带来的最大变化是：**函数式数据处理模式成为一等公民**。以前需要写大量`for`循环的场景，现在可以用迭代器+高阶函数的组合来实现：

```go
// Go 1.x: 需要显式循环和条件判断
var active []User
for _, u := range users {
    if u.Active {
        active = append(active, u)
    }
}

// Go 2.0: 使用迭代器组合
active := slices.Collect(
    slices.Filter(
        slices.Values(users), 
        func(u User) bool { return u.Active },
    ),
)
```

根据Go团队的数据，使用迭代器API后，典型的数据处理代码行数减少了30%，可读性显著提升。

## 错误处理改进

Go 2.0对错误处理进行了关键改进，同时保持了"显式错误处理"的哲学：

### 错误包装增强

```go
// 新的 errors.Join 和 errors.Unwrap 支持多错误
err1 := errors.New("file not found")
err2 := errors.New("permission denied")
combined := errors.Join(err1, err2)

// errors.Is 和 errors.As 支持多错误
if errors.Is(combined, os.ErrNotExist) {
    // 会匹配到 err1
}
```

### fmt.Errorf 增强

`fmt.Errorf`现在支持`%w`包装多个错误：

```go
return fmt.Errorf("process file %s: %w, %w", filename, err1, err2)
```

## 结构化日志：slog包

Go 2.0将`slog`（结构化日志）正式纳入标准库，结束了Go社区日志库碎片化的历史。

```go
import "log/slog"

func main() {
    logger := slog.New(slog.NewJSONHandler(os.Stdout, nil))
    logger.Info("user login",
        slog.String("user", "alice"),
        slog.Int("attempt", 3),
        slog.Duration("latency", 150*time.Millisecond),
    )
}
```

`slog`的设计哲学：
- 零依赖、高性能（比zap慢约10%，但使用更简单）
- 支持Text和JSON两种输出格式
- 支持日志级别（Debug、Info、Warn、Error）
- 支持结构化字段和属性分组
- 与OpenTelemetry集成，支持分布式追踪

## 泛型增强

Go 2.0在Go 1.18引入的泛型基础上进行了重大增强：

### 泛型类型推断改进

```go
// Go 1.18: 需要显式指定类型参数
result := slices.SortFunc[User](users, compareByName)

// Go 2.0: 类型参数自动推断
result := slices.SortFunc(users, compareByName)
```

### 泛型别名

Go 2.0支持泛型类型别名：

```go
type Set[E comparable] = map[E]struct{}
type Stack[E any] = []E
```
